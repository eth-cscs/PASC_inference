GitHub and doxygen

INITIALIZATION of gh-pages and first HTML page on github
-------------------------------------------------------------------------
- clone repository
git clone https://github.com/lukaspospisil/pizza.git
- create gh-pages branch (there will be stored html pages for git)
git checkout --orphan gh-pages
- delete all content in git repository (in active branch, i.e. in gh-pages)
git rm -rf .
- create sample page
echo "Hello world, this is our html fun!" > index.html
- add new file to git
git add index.html
- commit changes
git commit -a -m "First pages commit"
- push changes (but you deleted .git stuff from the folder, therefore it has to be set up again)
git push --set-upstream origin gh-pages
git push origin gh-pages
- open a web browser and test it:
http://lukaspospisil.github.io/pizza/
- from official Git documentation:
'Note that Pages are always publicly accessible when published, even if their repository is private.'


DOXYGEN for generating HTML
-------------------------------------------------------------------------
- create folder for documentation
mkdir html
- add html directory to .gitignore
